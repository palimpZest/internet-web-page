Lesson 0: How the Internet Works

Internet and HTML

The World Wide Web is the sum of all Web Pages. These are documents written in HTML (Hypertext Markup Language). They are connected and form a web thanks to Hyperlinks.

All types of files can be found on the web such as plain text, videos, images, music, and more. HTML is made of text content, markup, references to other documents and links to other pages.

Web browsers and HTTP

A Web browser allows to display those documents on a computer by making requests to the servers through the Internet (the world's largest network).

These requests use the HTTP protocol to communicate with servers, which are bigger versions of a computer and host the files.

Lesson 1: Introduction to HTML

HTML Markup

Markup is a text that is surrounded by an opening tag and a closing tag. A tag is contained in brackets. Some don't require a closing tab. These are called "void" tags. An element is everything within a set of opening and closing tags.

Computers are stupid

If a programmer doesn't write code carefully following the syntax, the computer might not understand his instructions. Computers are sensitive to typos and mistakes.

That is why many problems could arise if a tag is not closed. For instance, everything will be italicised after an (emphasis tag) if it is not closed.

Attributes, inlines and blocks

An attribute is a property of an HTML element. A tag can have multiple attributes. For example, the anchor tag allows to link to another URL.

There are two sorts of tags: inline and blocks. The Break tag and the Image tag are void tags because they have no content.

Structure of HTML

HTML uses indentations to make a piece of code more coherent and structured.

Indentation represents a new idea. In HTML, it increases when an idea begins and decreases when this idea is finished. In order to succesfully display a Web page, it is also important to explain how a piece of programming code works for another person to quickly understand it.

Also, programmers can't remember everything. If they forget a detail they can look it up.

Lesson 2: HTML and CSS Languages

A website like a house

  • If we use the analogy of a website as a house, then:
  • - HTML files are like its structure (walls, rooms, kitchen, etc.)
  • - CSS files are like the style of your house (colors, decoration, etc.)
  • - Javascript files are like the interactive components (tv remote, garage-door opener, etc.)

Tree-house structure

The code of a Web page is a Tree-like structure. When programmers talk about the DOM (Document Object Model) they are talking about this tree-like structure, which comes from the fact that the elements can have other elements inside of them. The more indented an element is, the more deeply nested its corresponding "box" is.

A collection of boxes

A web page is full of invisible boxes everywhere. Everything on a web page is actually a box but it is possible to change the shapes of the elements using styles.

Thinking about how to arrange and use those boxes is called Boxifying Design.

Developer Tools

The Developer Tools option in Google is useful. It allows to vizualize your code, to make changes to it, and test it without saving the files.

Lesson 3: CSS style and HTML structures

Coding better

It is important to avoid repetition to write good code! There are some benefits to this. It allows to save time, it helps to avoid mistakes and/or omissions. It is important to work more efficiently and to write more readable code.

Commenting the code allows to communicate to others and helps you also to explain what you are doing. In CSS, code comments begin with /* and end with */

Cascading Style Sheets

In CSS, a selector defines the elements of the page that will be applied a style. You can add style to a tag throughout the page or to a class, you can also combine selectors. CSS is like a search and replace tool.

  • Where you define the rules and in what order they are applied matters in CSS. The application of styles follows hierarchical rules. Styles are applied in the following order:
  • 1. the default style of a browser
  • 2. the stylesheet in a separate file
  • 3. stylesheet inside HTML
  • 4. inline style in an element

Also, rules are applied to the elements they match and to their child elements (within those elements). If a child's element has multiple, overlapping rules that apply to it, the most specific rule always takes effect.

Positionning boxes

Setting the box-sizing property to border-box makes the layout process easier (the size of the boxes can be set by a percentage instead of pixels).

One of the most interesting CSS's functions is the flex-box attribute. It helps positioning the boxes next to each other, so that a programmer can use all the space on a webpage.

  • 2. Look for respected styles and similar elements
  • 3. Write your HTML
  • 4. Apply your styles (from biggest to smallest)
